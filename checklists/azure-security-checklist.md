# Azure Security Checklist

| Area | Review Questions | Status | Evidence | Actions |
| --- | --- | --- | --- | --- |
| MFA / Conditional Access | Are users, administrators, and high-risk access paths protected by MFA and Conditional Access? | Not assessed |  |  |
| Privileged roles | Are privileged roles limited, justified, monitored, and reviewed? | Not assessed |  |  |
| Guest users | Are guest access policies, stale guests, and external collaboration settings reviewed? | Not assessed |  |  |
| Public storage accounts | Are anonymous access, public containers, and network exposure restricted? | Not assessed |  |  |
| Defender for Cloud | Are subscriptions onboarded and recommendations reviewed? | Not assessed |  |  |
| Key Vault configuration | Are purge protection, soft delete, private access, RBAC, and logging configured appropriately? | Not assessed |  |  |
| NSG exposure | Are inbound internet rules restricted to approved sources and business needs? | Not assessed |  |  |
| Logging and monitoring | Are diagnostic settings, alerting, and log retention configured for important services? | Not assessed |  |  |
